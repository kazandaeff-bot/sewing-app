**Важно:** Не хвалить просто так — ценить реальные достижения. Не критиковать без предложенных решений. Практические, понятные и применимые рекомендации.

## 💻 Платформа автоматизации швейного производства

### Стек
- Next.js 16 + Prisma + SQLite
- Запуск: `bun run dev` через `.zscripts/dev.sh` (иначе процесс убивается контейнером)
- URL: https://w1cem4d5ak91-d.space-z.ai/

### Что реализовано
- Продукты с размерами и цветами (drag-and-drop, удаление)
- Справочники: типы материалов, материалы, нормы расхода, заказчики, города
- API маршруты для всех сущностей
- Швея → ВТО → ОТК (на базе SewingTask с этапом утюжки, частичная сдача)
- Переделки (SewingRework)
- Авторизация (простая, НЕ JWT — задача на доработку)
- Вкладки: Швея, ОТК, Руководитель
- План пошива: автоназвание по заказчику, редактирование черновиков, возврат в черновик из «утверждён», дополнение позиций после раскроя
- Распределение по городам: автоподтягивание из планов, прогресс распределения
- Остатки кроя: автосоздание при actualQty > plannedQty
- Заказчики: самостоятельная регистрация, доступ только к своим данным (планы/города/короба/материалы)
- Детальный просмотр плана с прогрессом пошива (% отшито / % проверено ОТК)
- Задания швеям: каждое задание — отдельная позиция (независимая отправка на ВТО)
- ВТО этап: вкладка для ВТОшницы, швея → ВТО (pending_ironing) → ОТК (pending_qc), расчёт зарплаты
- Автосписание материалов: при отметке раскроя автоматически списываются ткани/фурнитура по нормам расхода
- Доступ заказчика к остаткам: галочка showMaterialBalance у заказчика, вкладка «Материалы» в личном кабинете

**Шапки для переключения (по запросу):**
- «наставник» → коучинг, анализ, приоритеты

- «технолог» → процессы, организация, оборудование

2. Чек-ин: как дела, что по плану, статус висящих задач
